Salesforce Administrator Portfolio

Salesforce CRM Implementation for US Healthcare SaaS Company

Role: Salesforce Administrator (Portfolio / Independent Project)

Duration: Dec 2025 – Present

Platform: Salesforce Sales Cloud & Service Cloud (Developer Edition)

Industry Focus: US Healthcare SaaS

1. Project Overview

This portfolio project demonstrates end-to-end Salesforce administration skills using a realistic US healthcare SaaS business scenario. The objective was to design, configure, and manage a scalable Salesforce CRM solution that supports sales operations, provider support, automation, data integrity, and reporting while following US best practices and healthcare-aware access controls.

2. Business Scenario

The business scenario represents a mid-sized US-based healthcare SaaS company that provides a patient engagement and provider management platform to hospitals and clinics.

Salesforce is used as the primary CRM system to manage:

  • Healthcare providers (Accounts)
  • Provider staff and patient liaisons (Contacts)
  • SaaS subscription sales (Opportunities)
  • Support cases and patient-related requests
  • Compliance-sensitive workflows

3. Business Challenges

  • Inconsistent data entry across sales and support teams
  • Manual processes causing delays and errors
  • Limited visibility into sales pipeline and support workload
  • Need for strict role-based access due to healthcare compliance requirements
  • Lack of standardized reporting for leadership

4. Business Requirements

Functional Requirements

  • Centralized CRM for sales and support operations
  • Automation to reduce manual effort
  • Role-based access control
  • Reliable and accurate reporting
  • Scalable configuration aligned with US healthcare operations

Non-Functional Requirements

  • Data integrity and validation
  • Audit-friendly configuration
  • Secure access model
  • Ease of use and user adoption

5. Salesforce Solution Overview

Salesforce was implemented using Sales Cloud and Service Cloud to meet business requirements. The solution focused on declarative configuration, automation using Flows, strong security controls, and actionable reporting.

6. Data Model Design

Standard Objects

  • Account: Healthcare providers (Hospitals, Clinics)
  • Contact: Provider staff and patient liaisons
  • Opportunity: SaaS subscription deals
  • Case: Support requests

Custom Object – Patient_Request__c

Purpose:

  • Track patient onboarding issues
  • Handle access and data correction requests
  • Manage compliance-related requests

Key Fields:

  • Request Type (Picklist)
  • Priority (Picklist)
  • Status (Picklist)
  • Due Date (Date)

Relationships:

  • Lookup to Account
  • Lookup to Contact

7. Security Model

Role Hierarchy

  • Sales Representative
  • Sales Manager
  • Support Agent
  • Compliance Officer
  • System Administrator

Profiles

  • Sales User
  • Support User
  • Compliance User

Permission Sets

  • Reporting Access
  • Patient Request Access
  • Compliance Read Only

Permission Sets were used to extend access instead of creating multiple profiles, following Salesforce and US enterprise best practices.

8. Automation Using Salesforce Flows

Flow 1: Opportunity Automation

  • Type: Record-Triggered Flow
  • Automatically updates Opportunity stage
  • Populates subscription start date
  • Validates required subscription fields

Flow 2: Patient Request Intake

  • Type: Screen Flow
  • Guided request creation for support users
  • Auto-assigns priority and ownership
  • Creates follow-up tasks

Flow 3: Case Escalation

  • Type: Record-Triggered Flow
  • Escalates cases open longer than defined SLA
  • Updates priority and notifies managers

9. Data Management & Quality

  • Imported sample data using Data Loader
  • Identified and resolved duplicate records
  • Performed data audits and cleanup
  • Enforced validation rules for required fields

10. Reports & Dashboards

Key Reports

  • Sales Pipeline by Provider Type
  • Open Patient Requests by Priority
  • Case Resolution Time
  • Data Quality Exception Report

Executive Dashboard – Healthcare Operations Dashboard

  • Sales pipeline visibility
  • Support workload overview
  • Compliance-related request tracking
  • Operational performance metrics

11. User Support & Testing

Simulated real-world admin scenarios:

  • User access issues
  • Flow execution errors
  • Validation failures

For each issue, root cause was identified, configuration or automation was corrected, and changes were tested and validated.

12. Documentation & Enablement

  • User onboarding guides
  • Flow usage documentation
  • Admin configuration notes

13. Key Learnings

  • Designing scalable security models is critical in regulated industries
  • Salesforce Flows can replace most legacy automation
  • Data quality directly impacts reporting and adoption
  • Clear documentation improves admin efficiency

14. Tools & Technologies Used

  • Salesforce Lightning Experience
  • Sales Cloud & Service Cloud
  • Record-Triggered and Screen Flows
  • Data Loader
  • Reports & Dashboards
  • Roles, Profiles, and Permission Sets

15. Conclusion

This portfolio project demonstrates hands-on Salesforce Administrator capabilities aligned with real-world US healthcare SaaS business requirements. It reflects end-to-end ownership of Salesforce configuration, automation, security, data management, and reporting, showcasing readiness for Salesforce Administrator and Business Systems roles in the US market.
